# Plan: Email Lifecycle Improvements for Akseli

Generated by /autoplan on 2026-04-12
Branch: master
Repo: JashanpreetSingh-dev/SimulatedTef

## Question
How should we upgrade the Akseli email system to be legally compliant and lifecycle-complete?

## Current State
- 3 emails total: `WelcomeToAkseli`, `SubscriptionCongrats`, `GoodFridayPromo` (one-off)
- Stack: Resend + React Email, idempotent via `userNotifications` MongoDB collection
- Queue-backed via BullMQ (`emailWorker.ts`)
- All emails are in French, no unsubscribe link, no click/open tracking
- No lifecycle emails beyond welcome + subscription congrats

## Scope of This Plan

### 1. Unsubscribe compliance (CASL)
Add a 1-click unsubscribe link to all transactional emails (welcome, subscription congrats) and all marketing emails (Good Friday, future promos).
- Add `unsubscribed_at` field to `userNotifications` or a new `emailPreferences` collection
- Add `POST /api/unsubscribe` endpoint (no auth required ã token-based)
- Pass unsubscribe URL into all email templates
- Guard all sends: check preference before sending

### 2. Day-3 no-session nudge
Send an email to users who signed up 3 days ago but have never completed a practice session.
- Trigger: BullMQ scheduled job, checks `results` collection for userId
- New email template: `Day3Nudge.tsx` ã "Vous n'avez pas encore commencûˋ" CTA to /practice
- Idempotent via `userNotifications` (type: `day3_nudge`)

### 3. Resend click tracking
Enable click tracking in all email sends so we can measure CTA conversion (welcome ã first session, congrats ã return session).
- Resend supports click tracking via `tags` and dashboard
- Add UTM parameters to all dashboard/CTA links in templates

### 4. Weekly progress digest
Send a weekly email summarizing the user's practice activity (sessions this week, scores, streak).
- Trigger: weekly BullMQ cron job (Monday morning)
- Pulls from `results` collection: count sessions in last 7 days, avg score, streak
- New email template: `WeeklyDigest.tsx`
- Respect unsubscribe preference
- Skip if user had 0 sessions AND 0 sessions the week before (lapsed users get win-back instead)
