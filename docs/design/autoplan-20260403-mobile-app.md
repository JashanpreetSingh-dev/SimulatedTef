# Plan: Should Akseli Have a React Native Android/iOS App?

Generated by /autoplan on 2026-04-03
Branch: master
Repo: JashanpreetSingh-dev/SimulatedTef

## Question
Should we build a React Native mobile app (iOS + Android) for Akseli — the TEF Canada oral practice platform?

## Current State
- Web app: React 19 + Vite, responsive design, deployed on Railway
- Core feature: Gemini Live oral examiner (real-time voice via WebSocket/PCM)
- Auth: Clerk
- Payments: Stripe
- Target user: Skilled immigrant, 2-3 months from TEF Canada exam
- Team: Solo founder (implied from codebase)
- Stage: Pre-significant traction, distribution sprint underway

## What a mobile app would add
- Native iOS + Android presence
- App store discoverability
- Push notifications for practice reminders
- Potentially better microphone access
- Add-to-home-screen experience

## What it costs
- React Native codebase: separate from web, shared logic possible but UI layer diverges
- App store review cycles (Apple: 1-7 days per release)
- Apple 30% / Google 15% subscription revenue cut
- Clerk + Stripe + Gemini Live audio all need RN-specific wiring
- Ongoing maintenance: two release pipelines, two codebases

### The Apple IAP Trap

If you ship to the App Store, Apple requires in-app purchases for digital subscriptions. You cannot link to Stripe from an iOS app. That means a completely separate billing system on top of everything else.
